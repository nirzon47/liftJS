JavaScript Declarations are Hoisted



In JavaScript, a variable can be declared after it has been used.

In other words; a variable can be used before it has been declared. 

x = 5
console.log(x);
var x;

What should console.log print?

  • 5
  • ReferenceError
Correct!

The declaration is hoisted so you can access it before line 4.

Incorrect!

The declaration is hoisted, even if it did not have any value, it would still not throw Reference Error.
